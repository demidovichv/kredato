# Compliance Policy — Email Marketing (RU/СНГ)

> Версия: 1.0 · Статус: ACTIVE · Обновлён: 2026-07-08
> Адаптировано из `AI_FINANCE_FACTORY/00_Project/11_COMPLIANCE_POLICY.md` (только чтение, исходник не изменён).
> Назначение: свод правил легальности для нашего email-проекта. Любая рассылка/сайт/оффер проходят проверку по этому документу.

---

## 🎯 Назначение

Определяет правила соблюдения законодательства РФ/СНГ, требований партнёрских платформ и внутренних этических стандартов.

Все действия агентов и изменения системы должны соответствовать настоящему документу.

---

## 📋 Обязательные проверки (для ВСЕХ задач)

- [ ] Соответствие законодательству (152-ФЗ о ПДн, 38-ФЗ + 347-ФЗ «О рекламе»/ЕРИР, КоАП за спам). Примечание: email- и push-рассылки освобождены от маркировки ЕРИР с сентября 2025, но opt-in + unsubscribe обязательны (КоАП: штрафы до 1 млн₽ юрлицам).
- [ ] Соответствие правилам партнёрских платформ (Leads.su, Pampadu, Statister).
- [ ] Отсутствие запрещённого контента.
- [ ] Отсутствие спама, манипуляций, вводящих в заблуждение действий.

---

## 🚫 Запрещённый контент (Forbidden)

Категорически запрещены:

| Категория | Примеры |
|-----------|---------|
| **Спам (холодная рассылка без согласия)** | Массовые письма без opt-in; Pampadu прямо запрещает (§12) → риск невыплаты |
| **Серые схемы** | Финансовые пирамиды, обман, фейковые отзывы |
| **Манипуляции** | Искусственное завышение метрик, накрутка подписчиков |
| **Нарушение авторских прав** | Чужой контент/картинки без разрешения |
| **Незаконный контент** | Наркотики, оружие, порнография, экстремизм |
| **Чужие перс. данные** | Публикация/передача ПДн без согласия |

## 📜 Правила платформ (по профилю нашего проекта)

### Leads.su (🟢 АКТИВЕН)
- Email **разрешён** как тип трафика (проверено: 10/11 вертикалей).
- Запрещён спам, фейк, бренд-снайпинг.
- Требуется честная лендинг-страница, соответствие офферу.
- За накрутку/мошенничество — бан и невыплата.

### Pampadu (🔴 ПАРКОВАН — не запускать без opt-in)
- Прямо запрещает холодный спам (§12 правил партнёрки).
- Требует **double opt-in** базу → иначе невыплата.
- Только RU/СНГ гео.
- Подключение только после готового opt-in ресурса (контакт @alexeyloktionov).

### Statister-Mail.ru (антифрод/мониторинг)
- Автостоп при аномалиях — не обходить защиту.
- Чистые базы, без «слитых» списков.

## ⚖️ Законодательство РФ/СНГ (критично с дня 1)

### 152-ФЗ (персональные данные)
- [ ] Политика конфиденциальности на сайте (`/privacy/`).
- [ ] Согласие на обработку ПДн (чекбокс, НЕ пречекнут).
- [ ] Фиксация согласия (IP + время).
- [ ] Unsubscribe в каждом письме.
- [ ] ИП как оператор ПДн (регистрация при необходимости).

### 38-ФЗ + 347-ФЗ (ЕРИР) — маркировка рекламы
- [ ] **Email- и push-рассылки освобождены от маркировки ЕРИР с сентября 2025** (digital-lawyer.ru, netology.ru) — маркировка НЕ требуется для email.
- [ ] Обязательны: opt-in база + механизм unsubscribe (one-click, заголовки `List-Unsubscribe`).
- [ ] КоАП: штрафы до 1 млн₽ юрлицам за спам/рассылку без согласия (152-ФЗ ПДн).
- [ ] Честные рекламные пометки, без скрытой рекламы (для иных каналов, где маркировка нужна — через ЕРИР/аккредитованного оператора).

### КоАП (спам)
- [ ] Рассылка только по opt-in базе.
- [ ] Холодная база — только если разрешено оффером И с соблюдением маркировки (у нас пока НЕТ таких офферов).

## 🔍 Compliance Review Process

1. **Агент/скрипт** создаёт изменение (письмо, лендинг, оффер, пост сайта).
2. **Проверка по чек-листу выше** (self-compliance для cron-агентов).
3. Если **PASS** → задача идёт дальше.
4. Если **FAIL** → блок, фиксация в `reports/compliance_violations.md`, возврат на доработку.

**SLA проверки:** стандарт < 2 ч, критичная публикация < 30 мин.

---

## 🚨 Эскалация нарушений

| Уровень | Действие |
|---------|----------|
| 1 нарушение (спринт) | Возврат на доработку, фиксация в логах |
| 2 нарушение | Анализ причины, предупреждение |
| 3 нарушение | Уведомление Owner (Вадим) |

Критическое (компрометация ключа, утечка ПДн, бан партнёрки) — немедленно Owner.
