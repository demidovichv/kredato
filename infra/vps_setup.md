# Инфраструктура: развёртывание VPS + изоляция через прокси

> Гео: **RU/СНГ на старт** (офферы RU/СНГ, аудитория — русскоязычная).
> Принцип: **ПК = только пульт**. Вся рассылючная машина живёт на VPS 24/7.
>
> ⚠️ **КРИТИЧНО (нерезидент):** владелец — НЕ резидент РФ, живёт в РБ. Аудитория — граждане РФ.
> → **152-ФЗ локализация ПДн**: база перс. данных граждан РФ **ДОЛЖНА храниться на территории РФ** (ст. 18 ч.5 152-ФЗ).
> → Значит **БАЗА (opt-in подписчики) → VPS в РФ** (или ЕАЭС/РБ с ЦОД в РФ), НЕ на EU-VPS.
> → Отправка писем — через SMTP-провайдера с РФ-серверами (Unisender Go / SendPulse / собственный Postfix на РФ-VPS).
> → Домены — зона **.com** (нерезидент не пройдёт Госуслуги с 1.09.2026 для .ru/.рф/.su). Выбран: **kredato.com** (Porkbun/Namecheap).
> → Выплаты от Leads.su/Pampadu — на карту РФ (решение владельца; риск валютного контроля НЕ блокирует старт).

## 🏗️ Архитектурная схема (скорректировано под 152-ФЗ)

```
[ТВОЙ ПК — тонкий клиент, РБ]
   ├─ SSH (через SOCKS-прокси) ──────> [VPS БАЗА / РФ локация]
   │                                    ├─ БАЗА подписчиков (ПДн граждан РФ → в РФ, локализация 152-ФЗ)
   │                                    ├─ майлер + прогрев (IP РФ-VPS)
   │                                    ├─ домены / DNS (kredato.com)
   │                                    ├─ Statister-агент (защита)
   │                                    └─ cron: monitor_all.py + автоматизация
   └─ Браузер (через RU-прокси) ─────> [HH.ru / VK / партнёрка / ящики]
                                        ↑ отдельный прокси НА АККАУНТ
```

**Почему VPS в РФ (а не EU, как планировалось изначально):**
- EU-VPS конфликтует с локализацией ПДн 152-ФЗ (хранение данных граждан РФ за рубежом = нарушение).
- РФ-VPS даёт и локализацию ПДн, и чистый IP для отправки в РФ-почтовики (Яндекс/Мail.ru доверяют РФ-IP больше).
- RU-прокси остаются для управления аккаунтами оператора (HH/VK/партнёрки).
- ⚠️ РФ-VPS могут быть под санкциями со стороны западных сервисов — но для RU/СНГ-рассылки это не критично.
- RU-айпишники часто под санкциями/блоками со стороны западных сервисов и самих рассылочных площадок
- EU-VPS даёт чистый IP-путь для отправки, RU-прокси — только для управления аккаунтами оператора
- Легче менять локацию VPS при необходимости (масштаб на EU/US позже)

## 2. Базовая настройка VPS (команды)

```bash
# Подключение (с ПК через прокси — см. раздел 4)
ssh root@<VPS_IP>

# Обновление
apt update && apt upgrade -y

# Создание не-root пользователя
adduser operator
usermod -aG sudo operator

# Настройка SSH-ключей (НЕ пароль!)
mkdir -p /home/operator/.ssh
# на ПК: ssh-copy-id operator@<VPS_IP>
chmod 700 /home/operator/.ssh

# Запрет root-логина по паролю
sed -i 's/PermitRootLogin yes/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
systemctl restart sshd

# Брандмауэр (порт 25 оставляем открытым для SMTP)
ufw allow 22,80,443,25,465,587
ufw enable

# Таймзона в RU для корректных логов
timedatectl set-timezone Europe/Moscow

# Установка базы
apt install -y python3 python3-pip git curl wget unzip
```

## 4. SSH через прокси (ПК → VPS)

Твой домашний IP не должен светиться при входе на VPS-воркер. Пускаем SSH через SOCKS-прокси:

**Вариант А — через SSH ProxyJump (если прокси поддерживает):**
```bash
# ~/.ssh/config на ПК
Host vps-worker
    HostName <VPS_IP>
    User operator
    ProxyCommand nc -X 5 -x <PROXY_HOST>:<PROXY_PORT> %h %p
```
Подключение: `ssh vps-worker`

**Вариант Б — через SOCKS в терминале:**
```bash
# Linux/Mac (или git-bash на Windows):
proxychains4 ssh operator@<VPS_IP>
# либо через PuTTY: Connection > Proxy > SOCKS5
```

⚠️ Важно: прокси для **управления VPS** может быть обычным (datacenter), потому что VPS — не рассылючный аккаунт. Отдельные **резидентные** прокси резервируем под HH.ru/VK/партнёрки (см. `proxies.md`).

## 5. Синхронизация vault (ПК ↔ VPS)

VPS пишет отчёты в `reports/`, ПК открывает их в Obsidian. Нужен двусторонний синк:

**Вариант А — Git (простой, если vault = git-репо):**
```bash
# На VPS после генерации отчёта:
cd ~/email_vault && git add -A && git commit -m "auto: digest" && git push

# На ПК:
git pull  # Obsidian увидит изменения
```

**Вариант Б — Syncthing (двусторонний, без облака):**
- VPS + ПК в одном Syncthing-кластере, папка `Email_Marketing_Repository` синхронизируется автоматически.
- Рекомендую — Obsidian не спотыкается, нет конфликтов как у git.

**Вариант В — Obsidian Sync (платно, но родное):** если не хочешь возиться.
