## 🔑 Управление секретами (Secrets Management)

### Хранение — ЗАПРЕЩЕНО
- API-ключи, пароли, токены, SMTP-креды в коде.
- Секреты в файлах, попадающих в Git/vault (открыто).

### Хранение — РАЗРЕШЕНО
- `secrets.local.md` (в `.gitignore`, НЕ в vault-синке).
- `.env` файлы локально (в `.gitignore`).
- Переменные окружения для передачи агентам.

> См. `secrets_template.md` — шаблон без реальных значений. Реальные креды пишутся ТОЛЬКО в `secrets.local.md`.

### Ротация
- [ ] Секреты обновляются не реже 1 раза в 90 дней.
- [ ] При компрометации — немедленный отзыв и замена.
- [ ] Процедуру ротации ведёт Hermes.

### Доступ
- [ ] К секретам — только агенты/скрипты, которым нужно.
- [ ] Доступ фиксируется в логах.
- [ ] Ежемесячный пересмотр прав.

## 📊 Классификация данных

| Уровень | Примеры | Требования |
|---------|---------|------------|
| **Public** | Опубликованные статьи, пины, контент сайта | Открытый доступ |
| **Internal** | Знания vault, метрики, конфиги | Только агенты + Owner |
| **Confidential** | Партнёрские офферы, стратегия, бизнес-планы | Строгий контроль, шифрование при передаче |
| **Restricted** | API-ключи, пароли, SMTP, ПДн подписчиков | Шифрование, строгий аудит |

---

## 🗄️ Безопасность данных

- **Vault (Obsidian):** содержит Internal + Confidential. Доступ только агентам/Owner.
- **data/:** сырые данные, логи — доступ для соответствующих агентов.
- **Git/vault-синк:** только аналитика и конфиги БЕЗ секретов. Protected.
- Все передачи — HTTPS/TLS. Передача секретов между агентами запрещена.

---

## 🖥️ Инфраструктурная безопасность

### Доступ
- [ ] К VPS/прокси/SMTP — только Owner (и Hermes в ограниченном объёме).
- [ ] MFA где возможно.
- [ ] Доступ в логах.

### Сеть
- [ ] Внешние соединения — TLS/HTTPS.
- [ ] Только доверенные API/сервисы.
- [ ] Брандмауэр (UFW на VPS) — входящий трафик ограничен.

### Бэкап и восстановление
- [ ] Бэкап vault + конфигов — не реже 1 раза в день (Syncthing/копия на VPS).
- [ ] Процедура восстановления задокументирована (см. [[vps_setup]]).

> См. `infra/proxies.md` — матрица прокси/аккаунтов (антрифрод, изоляция оператора).

---

## 🔍 Периодический аудит

- [ ] Hermes аудит безопасности ≥ 1 раз в месяц.
- [ ] Проверки: секреты в коде/логах, доступы, целостность данных, соответствие политикам.
- [ ] Ad-hoc аудит при: смене архитектуры, новом агенте, интеграции, после инцидента.

---

## 🚨 Реакция на инциденты

| Уровень | Пример |
|---------|--------|
| Minor | Секрет в логе (не актуальный) |
| Moderate | Утечка тестовых данных |
| Critical | Компрометация API-ключа, утечка ПДн подписчиков |

Процесс: Detection → Containment → Eradication → Recovery → Post-incident (фиксация в `reports/`).

Критические — немедленно Owner.
