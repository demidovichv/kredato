# Матрица прокси и аккаунтов

> Принцип: **оператор изолируется отдельными резидентными прокси**, рассылючная машина живёт на VPS.
> Прокси НЕ отправляют письма — они только для управления аккаунтами (HH.ru, VK, партнёрки, почтовые ящики).

## 🔑 Типы прокси (что берём, что нет)

| Тип | Для чего | Берём? |
|-----|----------|--------|
| **Residential (резидентные)** | Вход в RU-аккаунты (HH.ru, VK, партнёрки) | ✅ Обязательно |
| **Mobile (мобильные)** | То же + выше доверие платформ | ✅ Если бюджет позволяет |
| Datacenter | ❌ быстро в бан на RU-платформах | Нет (кроме SSH на VPS) |
| SOCKS5 | ✅ предпочтительнее HTTP (шире поддержка) | Да |

---

## 🗂️ Матрица аккаунт = прокси

Каждый аккаунт ЖЁСТКО привязан к своему прокси (антрифрод). Матрица — единственный источник правды.

| Аккаунт | Платформа | Тип прокси | Локация | Примечание |
|---------|-----------|-----------|---------|------------|
| op-hh-01 | HH.ru | Residential | RU | Чек-лист регистрации ИП (LeadMagnet) |
| op-hh-02 | HH.ru | Residential | RU | Запасной |
| op-vk-01 | VK (Senler) | Residential | RU | Связка HR→VK→Инфобиз |
| op-partner-01 | Партнёрка #1 | Residential | RU | Выплаты на ИП |
| op-mail-01 | Почтовый ящик (приём) | Residential | RU | Контроль спама |
| ssh-vps | VPS (SSH) | Datacenter/SOCKS | EU | Только управление машиной |

⚠️ **Правила:**
- НИКОГДА не заходить в аккаунт с другого прокси (сбросит антифрод-профиль)
- Менять прокси только если аккаунт «умер» и создаём новый
- Прокси на VPS-SSH может быть datacenter — это не рассылючный аккаунт

## 🛠️ Настройка на ПК (браузер через прокси)

Используем профили в браузере (например, Firefox Multi-Account Containers или отдельный профиль на прокси):

```
Профиль «HH-01» → SOCKS5 <proxy-ip-1>:<port>  (привязан к op-hh-01)
Профиль «VK-01» → SOCKS5 <proxy-ip-2>:<port>  (привязан к op-vk-01)
Профиль «Partner» → SOCKS5 <proxy-ip-3>:<port>
```

Не использовать один профиль для разных аккаунтов.

Для SSH на VPS (см. [[vps_setup]] раздел 4) — отдельный SOCKS, можно datacenter.
